Setting up the IDE


Wait. An IDE? What's that?
An IDE, or integrated development environment, is a way for programmers to easily write programs. To program an EV3, the most widely used IDE is the EV3 MINDSTORMS software. From now on, I'll refer to the LEGO MINDSTORMS software as an IDE.

How do I get started?
To get started, head over to the LEGO website and download the EV3 Education software. Install it, and start the program. You should be greeted with this starting screen.
Congratulations! You just set up the IDE!

Intro to Programming


How do I make a new program?
It's pretty easy, actually. All you need to do is click File, click New Project, and then double-click Program.

Programs are stored in files called projects, which can have logs, images, sounds, and more programs, all in a single file.

How does programing work?
Programing starts with a programming language. The IDE supports only one language, which is block-based. A block-based language means that you don't need to type a single word, which makes it a lot easier to learn. In a block-based language, there is a start, and then there are multiple functions which follow it, one after another. Each function chain is called a thread, which is a set of tasks the EV3 reads and executes. When the EV3 is multitasking, it is called multi-threading or parallel execution because there are multiple "threads" running at the same time. When you "run" the program, the IDE sends the program to the EV3, which the EV3 stores and then executes.

Thankfully, programming is really easy thanks to the good design of the IDE.

Blocks


Blocks? What are those?
Ah, blocks. They are the basis of the EV3 programming language. Remember the functions we were talking about earlier? Blocks are a representation of the functions within the EV3 programming language.

It's time to meet the blocks!

Action

World: meet program! Action blocks are how your program interacts with the physical world, either by moving motors, displaying an image, or playing a sound.

Flow

These blocks are the traffic signals of the programming world. They tell the program to stop, start, or repeat an action.

Sensor

Program: meet world! These blocks tell the program what's going on in the world, whether it be the color of the line below it, or the distance of its arm is to the wall.

Data Operator

These blocks take data and manipulate it. Use these blocks to multiply, divide, (calculate the sine of?) your data! In addition, you can use these blocks to store your data in variables.

Advanced

These blocks do niche stuff. They can access files for datalogging, stop the program, send bluetooth messages, and read raw sensor data. Don't mess with these if you don't know what you're doing!

MyBlock

These blocks can do anything! They can be used in tandem with other blocks to do anything. A thread can be compressed into a MyBlock and easily placed in a program.


Now that we know the main categories of blocks, let's take one apart to figure out it's anatomy!

The block pictured above is a color sensor block. We know it's a sensor block because the header is yellow, and we know that it interacts with the color sensor because it is pictured on the left side.
Currently, the block is set to see if it reads the same color code as 5, which is red. It outputs a boolean value (true or false) and a color code value (number 0-7).

Let's take a look at the different parts of the block.
  • The box in the top right corner is the port block. You can choose the port that the color sensor is connected to to target it.
  • The box in the bottom left side is the subfunction selector. By clicking this and choosing a different option, you can tell the function block to measure or compare a different value.
  • The 3 boxes to the right of the subfunction selector are the inputs and outputs of the block. Going from left to right:
    • The first one is an input because you can see indentations on the top. This input accepts an array object, which contains a list all of the different colors it is looking for.
    • The second box is an output because you can see indentations on the bottom. This box outputs whether or not the colors inputted match the color that the color sensor is looking at.
    • The final box outputs the color that the color sensor sees.

You can drag blocks to snap them together, like virtual legos. The outputs of one block can be dragged and snapped into another's block.

Blocks aren't that hard!

Sensors


What sensors are there? What do they do?
Here's a list of all the EV3 sensors, and what they do.

Touch Sensor


These sensors are the most basic sensors. They are essentially a button. They output a boolean value (true or false), and can be used as a way of detecting if your robot has hit a wall. Order

Color Sensor


These sensors are like the eyes of your robot. They can tell you the color, reflectivity, and the brightness of the surface they're pointed towards. Order

Ultrasonic Sensor


This sensor measures the distance between the sensor and the surface it is pointed towards. In addition, it can tell you if there are other sensors present. Order

Gyro Sensor


This sensor measures the its rotation around the vertical axis, relative to the world around the robot.
⚠ Warning! ⚠
This sensor has been known to be inaccurate. Avoid it unless absolutely necessary.
Order

Motor Rotary Encoder


This is a pseudo-sensor. Each EV3 motor has an embedded rotary encoder, which measures the rotation of the motor. You can access it and use it for measuring a motor's rotation. Order (large servo) Order (medium servo)

Infrared Sensor


This sensor measures proximity, the general direction of an infrared beacon, and it can recieve commands from an infrared beacon, like a remote-control car. Order (sensor) Order (beacon)

Variables


What are variables?
Variables are used to store data within a program. There are three types: numeric, text, and boolean. The number variable stores numbers, the text variable stores text (like a word), and the boolean type stores a conditional value, like true or false.

Let's take a look at an example of the main datatypes.

Number

Text

This datatype stores text, which is also known as a string. Any character you can type is valid here.
